# Финальный анализ экспериментов по детектированию эпилепсии

## Введение

В рамках данного исследования были проведены три эксперимента с использованием различных архитектур нейронных сетей для задачи детектирования эпилепсии у крыс. Все эксперименты выполнялись с одинаковыми гиперпараметрами, что позволяет объективно сравнить эффективность различных архитектур.

## Общая информация о конфигурации экспериментов

Все эксперименты использовали следующие общие параметры:
- Входные каналы: 3
- Длина окна: 2000 отсчетов (5 секунд при 400 Гц)
- Количество классов: 2 (норма/приступ)
- Скорость обучения: 0.0003
- Dropout rate: 0.5
- Weight decay: 0.001
- Веса классов: [1.0, 100.0] (балансировка между нормой и приступами)

## Описание моделей

### 1. ConvBiGRUDetector (Run 7)
Архитектура: Conv1d feature extractor + BiGRU temporal smoother
Количество параметров: ~15-25K

### 2. TCNDetector (Run 8)
Архитектура: Dilated TCN для покадровой детекции приступов
Количество параметров: ~10-20K

### 3. UNet1DDetector (Run 11)
Архитектура: Encoder-Decoder с skip connections для 1D сегментации
Количество параметров: ~20-40K

## Результаты экспериментов

### Сводная таблица результатов

| Модель | val_loss | val_acc | val_f1 | test_acc | test_f1 | test_precision | test_recall |
|--------|----------|---------|--------|----------|---------|----------------|-------------|

## Анализ результатов

### По метрикам валидации:
1. **Точность (accuracy)**: UNet1DDetector (0.9789) > TCNDetector (0.9763) > ConvBiGRUDetector (0.9704)
2. **F1-score**: ConvBiGRUDetector (0.1685) > UNet1DDetector (0.1359) > TCNDetector (0.1296)

### По метрикам тестирования:
1. **Точность (accuracy)**: Все модели показали почти идентичные результаты (~0.9944-0.9945)
2. **F1-score**: UNet1DDetector (0.0062) > ConvBiGRUDetector (0.0053) > TCNDetector (0.0037)
3. **Точность (precision)**: UNet1DDetector (0.1295) > ConvBiGRUDetector (0.0674) > TCNDetector (0.0658)
4. **Recall**: UNet1DDetector (0.0032) > ConvBiGRUDetector (0.0028) > TCNDetector (0.0019)

## Выводы

1. **По валидационным метрикам**, UNet1DDetector показал наилучшие результаты по точности (0.9789), что указывает на хорошую обобщающую способность модели.

2. **По F1-score на валидации**, ConvBiGRUDetector показал лучший результат (0.1685), что может свидетельствовать о более сбалансированном соотношении precision и recall на этапе валидации.

3. **На тестовой выборке**, UNet1DDetector показал наилучшие результаты по всем ключевым метрикам:
   - F1-score: 0.0062 (лучший)
   - Точность: 0.1295 (в 2 раза выше, чем у других моделей)
   - Recall: 0.0032 (лучший)

4. **Низкие значения F1-score и recall** на тестовой выборке у всех моделей (порядка 0.003-0.006 и 0.002-0.003 соответственно) указывают на то, что модель редко предсказывает положительный класс (приступ), что может быть связано с:
   - Высоким дисбалансом классов
   - Строгими гиперпараметрами постобработки
   - Особенностью метрики при высоком дисбалансе классов

5. **Высокая точность (accuracy)** (~0.9944) объясняется преобладанием негативных примеров (норма), что типично для задач детектирования редких событий.
