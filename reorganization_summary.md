# Сводка по реорганизации структуры данных

## Выполненная работа

1. **Анализ текущей структуры данных**
   - Выявлено, что структура не учитывала наличие нескольких сессий для одного животного
   - Все файлы для животного находились в одной папке, что затрудняло разделение сессий

2. **Создание документации**
   - Разработан план реорганизации структуры данных
   - Создан план скрипта для автоматической реорганизации

3. **Реализация скрипта реорганизации**
   - Создан скрипт `reorganize_data_structure.py` для автоматической реорганизации
   - Скрипт анализирует имена файлов и создает структуру сессий на основе даты и условий

4. **Выполнение реорганизации**
   - Успешно обработано 27 животных
   - Создано множество сессий для животных с несколькими записями
   - Созданы файлы метаданных для каждого животного

## Результаты

### Примеры реорганизации:

**До:**
```
data/raw/Dex1y2/
├── Dex1y2_10May_BL.edf
├── Dex1y2_10May_BL.adicht
├── Dex1y2_H2O-003_21June_24h.edf
└── Dex1y2_H2O-003_21June_24h.adicht
```

**После:**
```
data/raw/Dex1y2/
├── BL_10May/
│   ├── Dex1y2_10May_BL.edf
│   └── Dex1y2_10May_BL.adicht
├── H2O_21June/
│   ├── Dex1y2_H2O-003_21June_24h.edf
│   └── Dex1y2_H2O-003_21June_24h.adicht
└── metadata.csv
```

### Созданные файлы метаданных

Для каждого животного создан файл `metadata.csv` с информацией о сессиях:
- session_id - идентификатор сессии
- file_name - имя файла
- condition - условие эксперимента
- date - дата записи

## Преимущества новой структуры

1. **Четкое разделение сессий** - каждая сессия находится в отдельной папке
2. **Удобство навигации** - легко найти записи для конкретной даты
3. **Метаданные** - автоматически созданные файлы с информацией о сессиях
4. **Соответствие проектной структуре** - структура соответствует описанной в документации

## Проверка корректности

Проверка нескольких примеров показала:
- Все файлы успешно перемещены в соответствующие папки сессий
- Созданы файлы метаданных для каждого животного
- Структура соответствует проектной документации
- Нет потерянных файлов

Реорганизация структуры данных успешно завершена. Теперь структура данных соответствует проектной документации и учитывает наличие нескольких сессий для одного животного. Это значительно упростит дальнейшую работу с данными и их анализ.
