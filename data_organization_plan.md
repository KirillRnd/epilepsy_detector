# План реорганизации структуры данных

## Проблема текущей структуры

В текущей структуре данных не учтено, что на каждое животное может быть несколько записей в разные даты. Все файлы для одного животного находятся в одной папке, что не позволяет корректно разделить сессии.

Например, у животного Dex1y2 есть две сессии:
- Запись от 10 мая 2023 года (Dex1y2_10May_BL.edf)
- Запись от 21 июня 2023 года (Dex1y2_H2O-003_21June_24h.edf)

Аналогично, у Ati5x1 есть несколько сессий:
- Запись от 4 апреля (Ati5x1_BL_4April_24h.edf)
- Запись от 15 мая (Ati5x1_BL_15May.edf)

## Целевая структура

Для корректной организации данных необходимо создать следующую структуру:

```
data/
├── raw/
│   ├── <animal_id>/
│   │   ├── <session_id>/
│   │   │   ├── recording.edf
│   │   │   ├── recording.adicht
│   │   │   └── annotations.txt
│   │   └── metadata.csv
├── processed/
└── labels/
```

Где:
- `<animal_id>` - идентификатор животного (например, Dex1y2, Ati5x1)
- `<session_id>` - идентификатор сессии, формируемый на основе даты и условия эксперимента

## Правила формирования session_id

Для формирования идентификатора сессии будем использовать следующие правила:

1. **Основа названия**: Условие эксперимента + дата
   - BL_10May (базовая линия, 10 мая)
   - H2O_21June (вода, 21 июня)
   - estrus_15May (эструс, 15 мая)

2. **Формат даты**: Сокращенное название месяца на английском языке
   - Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec

3. **Условия эксперимента**:
   - BL - базовая линия
   - H2O - вода (контроль)
   - estrus, diestr, proestr, metestr - фазы цикла
   - Другие условия по префиксу в названии файла

## Примеры реорганизации

### До реорганизации:
```
data/raw/Dex1y2/
├── Dex1y2_10May_BL.edf
├── Dex1y2_10May_BL.adicht
├── Dex1y2_H2O-003_21June_24h.edf
└── Dex1y2_H2O-003_21June_24h.adicht
```

### После реорганизации:
```
data/raw/Dex1y2/
├── BL_10May/
│   ├── Dex1y2_10May_BL.edf
│   └── Dex1y2_10May_BL.adicht
├── H2O_21June/
│   ├── Dex1y2_H2O-003_21June_24h.edf
│   └── Dex1y2_H2O-003_21June_24h.adicht
└── metadata.csv
```

## Алгоритм реорганизации

1. Для каждого животного определить все сессии по именам файлов
2. Создать папки для каждой сессии
3. Переместить соответствующие файлы в папки сессий
4. Создать файл metadata.csv для каждого животного с информацией о сессиях
5. Обновить общую таблицу метаданных проекта

## Обновление метаданных

После реорганизации необходимо обновить:
1. Общую таблицу метаданных data/raw/metadata.csv
2. Метаданные для каждого животного data/raw/<animal_id>/metadata.csv
3. Документацию проекта при необходимости

## Проверка корректности

После реорганизации необходимо проверить:
1. Все файлы перемещены в правильные папки
2. Нет потерянных файлов
3. Структура соответствует описанной
4. Метаданные корректно обновлены
