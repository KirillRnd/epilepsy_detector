# Обзор системы детектирования эпилепсии у крыс

## 1. Общее описание проекта

Проект по созданию детектора эпилепсии у крыс включает в себя полный цикл разработки системы машинного обучения для анализа ЭЭГ/ЭКоГ записей у крыс. Система предназначена для автоматического обнаружения эпилептических приступов в записях ЭЭГ/ЭКоГ, сохраненных в форматах .adicht (LabChart) и .edf (European Data Format).

### 2.1. Описание основных директорий

#### data/raw/
Содержит исходные необработанные данные:
- Каждое животное в отдельной папке с идентификатором (например, `Dex1y2`)
- Каждая сессия записи в отдельной подпапке (например, `BL_10May`)
- Файлы записи (.edf или .adicht) и файлы разметки (.txt)

#### data/processed/
Содержит предобработанные данные:
- Предобработанные сигналы в формате .npy для быстрой загрузки
- Бинарные маски приступов в формате .npy
- Информация о сегментах наличия/отсутствия эпилепсии
- Метаданные датасета

#### data/labels/
Содержит файлы разметки:
- Текстовые файлы с разметкой приступов для каждой сессии
- Сводная таблица всех приступов

#### src/
Исходный код проекта:
- `data_loading/` - модули для загрузки данных
- `preprocessing/` - модули для предобработки сигналов
- `modeling/` - реализация моделей
- `evaluation/` - модули для оценки качества
- `utils/` - вспомогательные функции

#### notebooks/
Jupyter ноутбуки для исследовательского анализа и визуализации

#### experiments/
Конфигурации и результаты экспериментов:
- Каждый эксперимент в отдельной папке
- Конфигурационные файлы
- Логи обучения
- Сохраненные модели

#### results/
Результаты экспериментов и метрики

#### docs/
Документация проекта

#### requirements.txt
Зависимости проекта

## 3. Основные компоненты системы

### 3.1. Загрузчик данных
- Поддержка форматов .edf и .adicht
- Унифицированный интерфейс для чтения данных
- Интеграция с системой метаданных

### 3.2. Модуль чтения разметки
- Парсинг текстовых файлов с разметкой приступов
- Поддержка различных форматов разметки
- Создание бинарных масок для обучения

### 3.3. Предобработка сигналов
- Фильтрация (полосовой и режекторный фильтры)
- Ресемплинг к единой частоте дискретизации
- Нормализация амплитуд
- Обнаружение и обработка артефактов
- Создание сегментов наличия/отсутствия эпилепсии

### 3.4. Балансировка классов
- Undersampling нормальных примеров
- Аугментация примеров с приступами
- Взвешивание классов в функции потерь

### 3.5. Модель детектирования
- 1D сверточная нейронная сеть
- Базовая и улучшенная архитектуры
- Поддержка остаточных блоков

### 3.6. Экспериментальный фреймворк
- План экспериментов по оптимизации модели
- Кросс-валидация по животным
- Комплексная оценка качества

## 4. Технические характеристики

### 4.1. Поддерживаемые форматы:
- .edf (European Data Format)

### 4.2. Основные библиотеки:
- mne - для обработки нейрофизиологических данных
- pyedflib - для низкоуровневой работы с EDF файлами
- neo - для чтения проприетарных форматов
- PyTorch - для реализации нейронных сетей
- scikit-learn - для классических методов ML и оценки качества

### 4.3. Рекомендуемые параметры:
- Частота дискретизации: 400 Гц
- Количество каналов: 1-4
- Баланс классов: 25% приступов, 75% нормальной активности

## 5. План реализации

### Этап 1: Подготовка данных (1-2 недели) (DONE)
1. Реализация загрузчиков для .edf файлов
2. Создание модуля чтения разметки приступов
3. Разработка скриптов для конвертации .edf
4. Подготовка метаданных в формате CSV

### Этап 2: Предобработка и подготовка датасета (1-2 недели)
1. Реализация модуля загрузки данных (предварительно изучить data/processed/conversion_summary.csv)
2. Разработка стратегии балансировки классов
3. Подготовка обучающего, валидационного и тестового датасетов

### Этап 3: Разработка модели (2-3 недели)
1. Реализация базовой TCN (Temporal Convolution Network) архитектуры
2. Создание системы для оценки качества модели
3. Обучение и оптимизация модели

### Этап 4: Валидация и тестирование (1-2 недели)
1. Проведение серии экспериментов
2. Кросс-валидация по животным
3. Анализ ошибок и чувствительности модели
4. Подготовка финального отчета

## 6. Ожидаемые результаты

### Метрики качества:
- Accuracy: > 90%
- F1-Score для класса приступов: > 0.8
- Sensitivity (Recall): > 0.85
- Specificity: > 0.95
- AUC-ROC: > 0.95

### Технические характеристики:
- Время обработки одного сегмента: < 10 мс
- Требования к памяти: < 1 ГБ
- Поддержка работы в реальном времени

## 7. Возможные улучшения

### Краткосрочные:
1. Добавление поддержки других форматов (BIDS, EEGLAB)
2. Расширение набора аугментаций для ЭЭГ данных
3. Реализация ансамблей моделей
4. Добавление механизма внимания

### Долгосрочные:
1. Self-supervised pretraining на больших объемах нормальных данных
2. Комбинирование CNN с рекуррентными слоями (LSTM/GRU)
3. Использование трансформеров для анализа временных последовательностей
4. Разработка системы онлайн-мониторинга приступов

## 8. Заключение

Разработанная архитектура системы детектирования эпилепсии у крыс предоставляет комплексное решение для автоматического анализа ЭЭГ/ЭКоГ записей. Система включает все необходимые компоненты для полного цикла обработки данных - от чтения исходных файлов до получения предсказаний модели.

Предложенная архитектура является модульной и расширяемой, что позволяет легко добавлять новые функции и улучшать отдельные компоненты системы. План экспериментов обеспечивает систематический подход к оптимизации модели и получению воспроизводимых результатов.

Следуя предложенному плану реализации, можно создать эффективную систему детектирования эпилептических приступов у крыс, которая будет полезна для нейробиологических исследований.
