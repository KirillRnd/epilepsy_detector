# Детали реализации системы детектирования эпилепсии у крыс

## 1. Загрузка данных

### 1.1. Загрузчик данных для .edf файлов

Загрузчик данных для .edf файлов реализован в классе `EDFLoader` в модуле `src/data_loading/edf_loader.py`.

Основные функции класса:
- `__init__(self, preload_data: bool = False)` - инициализация загрузчика с возможностью предзагрузки данных
- `load_file(self, file_path: str) -> Dict` - загрузка .edf файла и извлечение метаданных
- `get_data(self, file_path: str, channels: Optional[List[str]] = None) -> np.ndarray` - получение данных из загруженного файла
- `get_channel_names(self, file_path: str) -> List[str]` - получение имен каналов из файла
- `get_sampling_frequency(self, file_path: str) -> float` - получение частоты дискретизации из файла
- `close_file(self, file_path: str) -> None` - освобождение ресурсов, связанных с файлом
- `close_all(self) -> None` - освобождение всех ресурсов

### 1.2. Расширенный загрузчик с поддержкой предобработки

Расширенная реализация загрузчика с поддержкой предобработки реализована в классе `AdvancedEDFLoader` в модуле `src/data_loading/edf_loader.py`.

Основные функции класса:
- `__init__(self, preload_data: bool = False, target_sampling_rate: Optional[float] = None, channel_selection: Optional[List[str]] = None)` - инициализация расширенного загрузчика
- `load_and_preprocess(self, file_path: str, apply_filter: bool = True, low_freq: float = 1.0, high_freq: float = 100.0, apply_notch: bool = True, notch_freq: float = 50.0) -> Dict` - загрузка и предварительная обработка .edf файла
- `extract_windows(self, file_path: str, window_length: float = 5.0, window_overlap: float = 0.0, preprocess: bool = True) -> Tuple[np.ndarray, List[Dict]]` - извлечение временных окон из файла

### 1.3. Загрузчик с интеграцией системы метаданных

Загрузчик с интеграцией системы метаданных реализован в классе `IntegratedEDFLoader` в модуле `src/data_loading/edf_loader.py`.

Основные функции класса:
- `__init__(self, metadata_file: str, **kwargs)` - инициализация загрузчика с метаданными
- `get_file_metadata(self, file_path: str) -> Dict` - получение метаданных файла из таблицы
- `load_with_metadata(self, file_path: str) -> Dict` - загрузка файла с интеграцией метаданных

## 2. Чтение разметки приступов

### 2.1. Базовый модуль для чтения разметки

Модуль для чтения разметки приступов реализован в классе `SeizureAnnotationReader` в модуле `src/data_loading/seizure_annotation_reader.py`.

Основные функции класса:
- `__init__(self)` - инициализация модуля
- `load_annotation_file(self, file_path: str, file_id: Optional[str] = None) -> Dict` - загрузка файла разметки приступов
- `get_seizures(self, file_id: str) -> List[Dict]` - получение списка приступов для файла
- `get_seizure_dataframe(self, file_id: str) -> pd.DataFrame` - получение разметки приступов в виде DataFrame
- `get_all_seizures_dataframe(self) -> pd.DataFrame` - получение разметки всех приступов в виде одного DataFrame
- `get_seizure_count(self, file_id: str) -> int` - получение количества приступов в файле
- `get_total_seizure_duration(self, file_id: str) -> float` - получение общей длительности приступов в файле
- `clear_annotations(self, file_id: Optional[str] = None) -> None` - очистка загруженной разметки

### 2.2. Расширенный модуль для чтения разметки

Расширенный модуль для чтения разметки приступов с поддержкой различных форматов реализован в классе `AdvancedSeizureAnnotationReader` в модуле `src/data_loading/seizure_annotation_reader.py`.

Основные функции класса:
- `__init__(self)` - инициализация расширенного модуля
- `load_annotation_file(self, file_path: str, file_id: Optional[str] = None, format_type: str = 'simple') -> Dict` - загрузка файла разметки приступов с указанием формата
- Поддержка различных форматов: simple, bids, eeglab, custom

### 2.3. Интегрированный модуль для чтения разметки

Модуль для чтения разметки с интеграцией с загрузчиком данных реализован в классе `IntegratedAnnotationReader` в модуле `src/data_loading/seizure_annotation_reader.py`.

Основные функции класса:
- `__init__(self, data_loader=None)` - инициализация интегрированного модуля
- `get_seizures_for_recording(self, file_id: str, recording_duration: Optional[float] = None) -> List[Dict]` - получение разметки приступов с проверкой корректности временных меток
- `create_binary_mask(self, file_id: str, recording_duration: float, sampling_rate: float) -> np.ndarray` - создание бинарной маски для разметки приступов
- `get_seizure_windows(self, file_id: str, window_length: float = 5.0, overlap_ratio: float = 0.0) -> List[Dict]` - получение информации о временных окнах с приступами

## 3. Предобработка сигналов

### 3.1. Функции предобработки

Комплексный класс для предобработки сигналов ЭЭГ реализован в классе `SignalPreprocessor` в модуле `src/preprocessing/signal_preprocessor.py`.

Основные функции:
- `apply_bandpass_filter` - применение полосового фильтра к сигналу
- `apply_notch_filter` - применение режекторного фильтра для подавления сетевых помех
- `resample_signal` - ресемплинг сигнала к целевой частоте дискретизации
- `z_normalize` - Z-нормализация данных
- `minmax_normalize` - минимакс нормализация данных
- `detect_amplitude_artifacts` - обнаружение артефактов по превышению порога амплитуды
- `remove_artifacts` - удаление артефактов из данных
- `create_sliding_windows` - создание скользящих окон из сигнала

### 3.2. Класс SignalPreprocessor

Класс `SignalPreprocessor` предоставляет комплексную предобработку сигнала:
- `__init__` - инициализация предобработчика с параметрами
- `preprocess` - комплексная предобработка сигнала
- `preprocess_window` - предобработка отдельного временного окна

### 3.3. Рекомендуемые параметры предобработки

```python
PREPROCESSING_CONFIG = {
    'target_sampling_rate': 512.0,
    'bandpass_low': 1.0,
    'bandpass_high': 100.0,
    'notch_freq': 50.0,
    'normalize_method': 'z_score',
    'artifact_removal': True,
    'artifact_threshold': 5.0,
    'window_length': 5.0,  # секунды
    'window_overlap': 0.5   # 50% перекрытие
}
```

## 4. Балансировка классов

### 4.1. Функции балансировки

Модуль для балансировки классов реализован в файле `src/preprocessing/class_balancer.py`.

Основные функции:
- `random_undersampling` - случайный undersampling для балансировки классов
- `complexity_based_undersampling` - андерсэмплинг на основе сложности
- `smote_oversampling` - SMOTE oversampling для генерации синтетических примеров
- `augment_eeg_data` - аугментация данных ЭЭГ для балансировки классов
- `calculate_class_weights` - вычисление весов классов для балансировки
- `balance_dataset_for_epilepsy_detection` - балансировка датасета для детектирования эпилепсии

### 4.2. Рекомендуемые параметры балансировки

```python
BALANCING_CONFIG = {
    'seizure_ratio': 0.25,  # 25% приступов в сбалансированном датасете
    'augmentation_factor': 3,  # Увеличить minority класс в 3 раза
    'noise_level': 0.01,      # Уровень шума для аугментации
    'undersampling_method': 'random',  # Метод undersampling
    'use_class_weights': True,  # Использовать веса классов
    'class_weight_method': 'balanced'  # Метод вычисления весов
}
```

### 4.3. Оценка эффективности балансировки

Функция `evaluate_balancing_effectiveness` предоставляет оценку эффективности балансировки классов с различными метриками:
- Sensitivity и Specificity
- Precision и Recall для класса приступов
- F1-мера
- Сбалансированная точность
- ROC AUC

## 5. Модель детектирования

### 5.1. Простой 1D-CNN детектор

Простой 1D-CNN детектор эпилептических приступов реализован в классе `SimpleEEGDetector` в модуле `src/modeling/simple_cnn_detector.py`.

Архитектура модели:
- Три сверточных слоя с нормализацией и пулингом
- Три полносвязных слоя с dropout
- Поддержка настраиваемого количества каналов и длины окна

### 5.2. Улучшенный 1D-CNN детектор

Улучшенный 1D-CNN детектор с остаточными блоками реализован в классе `ImprovedEEGDetector` в модуле `src/modeling/simple_cnn_detector.py`.

Архитектура модели:
- Начальные сверточные слои
- Остаточные блоки с увеличением количества каналов
- Адаптивный пулинг для фиксированного размера
- Полносвязные слои с dropout

### 5.3. Функции обучения и оценки

Модуль предоставляет функции для полного цикла обучения и оценки:
- `weighted_cross_entropy_loss` - взвешенная кросс-энтропия для балансировки классов
- `train_epoch` - обучение модели в течение одной эпохи
- `validate_epoch` - валидация модели
- `train_model` - полный цикл обучения модели
- `evaluate_model` - оценка качества модели на тестовой выборке

### 5.4. Рекомендуемые параметры модели


## 6. Стратегия обработки данных

### 6.1. Стратегия чтения файлов .adicht и .edf

#### Обработка .edf файлов
- Используемые библиотеки: mne (основная), pyedflib (низкоуровневая)
- Рекомендуемый подход: использование mne как основной библиотеки

#### Обработка .adicht файлов (LabChart)
- Стратегии обработки:
  1. Использование специализированных библиотек (neo)
  2. Конвертация в стандартные форматы (.edf)
- Рекомендуемый подход: конвертация в .edf с помощью LabChart

### 6.2. План предобработки сигналов ЭЭГ

План предобработки включает:
1. Фильтрация сигналов (полосовой и режекторный фильтры)
2. Ресемплинг к единой частоте дискретизации
3. Нормализация амплитуд
4. Обнаружение и обработка артефактов
5. Сегментация сигналов на временные окна

### 6.3. Стратегия балансировки классов

Стратегия балансировки для проекта детектирования эпилепсии:
1. Комбинированный подход:
   - Undersampling для majority класса (нормальная активность)
   - Аугментация для minority класса (приступы)
   - Взвешивание классов в функции потерь
2. Рекомендуемые параметры балансировки
