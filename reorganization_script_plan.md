# План скрипта для реорганизации структуры данных

## Общее описание

Скрипт должен реорганизовать структуру данных в папке `data/raw`, перемещая файлы в правильные папки сессий для каждого животного.

## Основные шаги

1. **Анализ текущей структуры**
   - Просмотр всех файлов в `data/raw`
   - Группировка файлов по идентификатору животного
   - Определение сессий для каждого животного на основе имен файлов

2. **Создание структуры сессий**
   - Для каждого животного определить уникальные сессии
   - Создать папки для каждой сессии
   - Извлечь информацию о дате и условии из имени файла

3. **Перемещение файлов**
   - Переместить файлы в соответствующие папки сессий
   - Обработка конфликтов имен
   - Сохранение оригинальных имен файлов

4. **Создание метаданных**
   - Создать файл metadata.csv для каждого животного
   - Заполнить метаданные на основе информации из README файла

## Алгоритм определения сессий

Для каждого файла извлекать информацию:
- Дата записи (из имени файла)
- Условие эксперимента (BL, H2O, estrus и т.д.)
- Фаза цикла (если указана)

Формировать session_id как комбинацию условий и даты.

## Обработка особых случаев

1. **Файлы с одинаковыми именами**
   - Добавлять суффикс при копировании
   - Логировать такие случаи

2. **Файлы без явной даты**
   - Использовать информацию из README
   - При необходимости запрашивать уточнение

3. **Неполные пары файлов**
   - Обрабатывать .edf и .adicht файлы отдельно
   - Логировать неполные наборы

## Структура выходных данных

```
data/raw/
├── <animal_id>/
│   ├── <session_id>/
│   │   ├── recording.edf
│   │   ├── recording.adicht
│   │   └── annotations.txt
│   └── metadata.csv
└── metadata.csv
```

## Требования к реализации

1. **Сохранность данных**
   - Не удалять оригинальные файлы до подтверждения успешности операции
   - Создавать резервные копии при необходимости

2. **Логирование**
   - Подробное логирование всех операций
   - Фиксация ошибок и предупреждений
   - Создание отчета о выполнении

3. **Обратимость**
   - Возможность отката изменений при необходимости
   - Создание файла с информацией о произведенных изменениях

## План тестирования

1. **Тест на одном животном**
   - Выбрать животное с несколькими сессиями
   - Проверить корректность создания структуры
   - Убедиться в сохранности всех файлов

2. **Тест на особых случаях**
   - Проверить обработку файлов без явной даты
   - Проверить обработку конфликтов имен
   - Проверить обработку неполных наборов файлов

3. **Полный тест**
   - Запустить на всех данных
   - Проверить структуру результата
   - Убедиться в отсутствии потерь данных

## Возможные улучшения

1. **Интеграция с README**
   - Использовать информацию из README для более точного определения сессий

2. **Валидация результатов**
   - Автоматическая проверка корректности структуры после реорганизации

3. **Откат изменений**
   - Создание скрипта для отката изменений при необходимости
